Temporal Trigger Engine Demo

Frame-rate independent trigger detection with microsecond precision

About This Demo

This demo shows the temporal trigger engine in action. The engine detects when rotating points cross the Y-axis, with sub-frame precision regardless of the actual animation frame rate.

Key features demonstrated:

  • High-resolution time slicing (1000Hz internal resolution)
  • Sub-frame crossing detection with Bresenham-inspired algorithm
  • Position interpolation for accurate crossing time calculation
  • Configurable cooldown time that works correctly at any frame rate
  • Reliable detection at varying rotation speeds

Frame-rate Dependent Issues (Original System)

  • Missed triggers at low frame rates
  • Fixed cooldown in milliseconds affected by frame rate
  • MAX_REASONABLE_MOVEMENT restrictions causing missed triggers
  • Debouncing logic skipping frames
  • Large angle jumps handled inconsistently

Frame-rate Independent Solutions (Temporal Engine)

  • High-precision time slice simulation (1000Hz)
  • Continuous path sampling between frames
  • Proper sub-frame interpolation for all crossings
  • Time-based cooldown with microsecond precision
  • Detection of all crossings, even during large angle jumps

How to Test Different Scenarios

  • Low Frame Rate: Set Frame Rate Limit to 15 FPS and observe consistent triggers
  • High Rotation Speed: Increase rotation speed to 3-5 rotations/sec
  • Cooldown Behavior: Set a cooldown of 100-300ms and observe how it works across any frame rate
  • Frame Rate Independence: Try changing frame rate limit while keeping rotation speed constant
